# Gate Footer Format — per decoding.py PHASE_FOOTER_SPEC

Each gate `open` command passes content parsed as `KEY: value` lines. These
fields are checked structurally by `decoding.py` when `QLN_BOOTSTRAP` is set.

## S → G (gate x)

```
X: <the question, a complete sentence ending in ?>
```

## G → Q (gate y)

```
ALPHA: <the irreducible essence, one clause, no "not">
SEEKS: <what α inherently moves toward — its self-interest>
```

## Q → P (gate z)

```
PHI: <the work's self-interest as grown — never anyone's perception>
OMEGA: <the field of universal interest it would live within>
ALIGNMENT: natural|partial|none|forced
EXTENT: <0-10, how far φ's seeking lies along Ω without forcing>
Z: <only what actually locked — omit this line entirely if ALIGNMENT is forced or none>
```

## P → V (gate a)

```
VALUE_MAX: <the greatest value this direction yields — the maximum, named concretely>
ENERGY: <what it costs / why it is unforced>
A: <the flow direction — where maximum value per energy points>
```

## V → next S (gate b)

```
L: <what crystallized here and now>
B2: <the artifact, carrying α>
INF0P: <the return question — ends in ?, not the seed repeated>
LIVENESS: <0-10, your honest attestation of how alive ∞0' is relative to the seed>
```
